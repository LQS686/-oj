系统架构总览

▼

DSOJ 采用 Next.js 16 全栈架构，前后端统一在一个工程内，通过 App Router 提供 SSR 页面与 Route Handler API。中间层由 Middleware 与 withApi 包装器协同处理鉴权、CSRF、限流与异常；业务层由 11 个 lib/ 模块组成；数据层基于 MongoDB（Prisma ORM）+ Redis（缓存/限流）+ BullMQ（评测/AI 队列）。

数据请求流程

📋 流程节点说明

1. 用户浏览器：React 组件通过 SWR 发起 fetch 请求，自动携带 httpOnly Cookie。

2. Next.js Middleware：拦截所有请求，校验 CSRF Token、应用限流策略、保护 /admin 路径。

3. withApi 包装器：统一鉴权（解析 Cookie 中的 JWT）、异常捕获、响应包装（ok/fail）。

4. Route Handler：使用类型安全的 ctx.params（Next.js 16 动态路由，已清理 85 处 as any）。

5. lib/ Service：11 个业务模块处理领域逻辑，调用 Prisma 进行数据库操作。

6. Prisma ORM → MongoDB：类型安全查询，38 个模型覆盖全部业务实体。

7. JSON 响应 → SWR：统一 ok()/fail() 响应格式，SWR 自动缓存与重新验证。

🔐 鉴权流程：JWT 存储于 httpOnly Cookie（非 localStorage），每次请求自动携带，Middleware + withApi 双重校验。ctx.params 已在 Next.js 16 动态路由下完全类型安全（50 个 route.ts 全部修复）。
